# Runbook index

Operators need tested procedures for deployment rollback, queue/DLQ triage and redrive, provider outage isolation, payment reconciliation, duplicate-effect investigation, database recovery, Redis degradation, credential rotation, PII incident containment, model rollback, and human-ticket backlog. Until those procedures are implemented and exercised, the corresponding automated capability remains gated or hands off safely. Trigger thresholds and owners are defined per environment, never in this index.
