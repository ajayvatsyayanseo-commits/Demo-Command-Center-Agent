# Problem definition

NXTutors has a captured website lead, a capable WhatsApp lead agent, an onboarding agent, and website payment/profile flows, but no single durable owner for the demo lifecycle. The result is disconnected identity, availability, confirmation, meeting, reminder, outcome, conversion, and payment state.

The Demo Command Center owns the lifecycle from a canonical demo handoff through paid onboarding acknowledgement. It does not own public WhatsApp ingress, legacy profiles/tutors/plans/subscriptions, payment truth outside verified Cashfree evidence, or arbitrary language-model decisions.

Success means one coherent user experience; atomic scheduling; exactly-once external effects; explainable, deterministic conversion scoring; policy-controlled messages/offers; server-verified payment; auditable website activation; and region-scoped operations. Failure must be explicit, retryable only when safe, or handed to a human.

This scaffold defines boundaries and contracts. It does not claim live provider readiness.
